What are arrays?#

Arrays are collections of elements that allow access and manipulation through their indices.

Bubble sort#

Bubble sort is perhaps the algorithm that feels most natural to many of us when sorting objects in real life. The Bubble sort algorithm works by sorting an array from the lowest to the highest values. Let’s imagine the following (I reckon, unimaginable) scenario:

You have a bookshelf with n books stored in random order. Each book has a unique thickness (the value), but they are randomly arranged on the shelf. Now, you’ve decided to sort the books from thinnest to thickest. How do you do this?

If you have a bookshelf nearby, you can apply the algorithm yourself in real life. Otherwise, I’ll provide the pseudocode (not the abstract version you often see) along with some illustrations:

  1. Start: Begin on the left of the bookshelf, at the position with index 0.
  2. Swap: Compare the book at index 0 with the book to its right (index 1). If the book at index 0 is thicker than the one at index 1, swap them so that the thicker book moves to index 1 and the thinner book stays at index 0. Next, compare the book at index 1 with the book at index 2 and perform the same check. Continue this process until you reach the last book at index n – 1. By the end of this pass, the thickest book will have “bubbled up” to the last position on the shelf. You have now completed one pass through the bookshelf.
  3. Optimize: For each subsequent pass, you can skip the last book, as it is already the thickest book from the previous pass.
  4. Repeat: Continue repeating the passes until all the books are sorted.

This method ensures that with each pass, the next thickest book settles into its correct position, gradually sorting the entire collection from thinnest to thickest.

If you want to see the more abstract pseudocode, I invite you to read the pseuodocode that you can find on Wikipedia or some other website on the internet.

Finally, we can implement the algorithm itself in C (with some illustrative comments in the book analogy):

void bubbleSort(int arr[], int n)
{
	// We iterate through each book in the shelf
	// starting at index 0 and moving to index (n - 2)
	// using the outer for loop. Why (n - 2)? Since we 
	// only need to look at the first (n - 1) indeces, and we start
	// counting at 0 to i <= n - 2 (or equivalently: i < n - 1),
	// will iterate through the first (n - 1) elements.
	for (int i = 0; i < n - 1; ++i)
	{
		// We include an indicator. Why do we do this? Simple:
		// once no swap occurs anymore (in the next for loop),
		// we can be sure that the "lower"-part of the array is
		// also sorted, so we don't need to pass over these parts
		// again.
		bool swapped = false; // Set to false at the beginning of each iter.

		// We use this for-loop to iterate and compare the books in the shelf
		// with excluding the last one at each pass, hence:
		// j < (n - 1) - i.
		for (int j = 0; j < (n - 1) - i; ++j)
			// If element at index j is higher than the index at
			// element j + 1, we swap them.
			if (arr[j] > arr[j + 1])
			{
				// Standard swap in C (using temp var.):
				int temp = arr[j];
				arr[j] = arr[j + 1];
				arr[j + 1] = temp;

				// If swapped, we set the indicator to true.
				swapped = true;
			}

		// If no swap has occured, we can break out of the for loop:
		if (!swapped)
			break;
	}
}

This is a simple explanation of the Bubble Sort algorithm. Regarding computational time complexity, the inner for loop runs \(n\) times to perform \(n\) comparisons. Additionally, the outer for loop also executes \(n\) times. As a result, the algorithm performs \(n \cdot n\) comparisons, leading to a time complexity of \(O(n \cdot n) = O(n^2)\). Note that time complexity typically represents the worst-case scenario; in practice, the algorithm may terminate earlier if the array becomes sorted before completing all iterations.

Selection sort#

Now that we’ve explored Bubble Sort (and hopefully gained an understanding of how it works), let’s turn our attention to the Selection Sort algorithm. This algorithm identifies the smallest value in an array and moves it to the front.

Using the same bookshelf analogy as above, the Selection Sort algorithm can be described as follows (in simple pseudocode):

  1. Start: Begin at the first book on the unordered shelf (index 0). Assume this book is the thinnest (i.e., has the lowest value).
  2. Iterate: Go through the entire bookshelf, one book at a time, searching for a thinner book. If a thinner book is found, update the current thinnest book. Continue until all books on the shelf have been checked.
  3. Move to the front: Once the thinnest book in the entire bookshelf is identified, swap it with the book at index 0. The book previously at index 0 moves to the position of the thinnest book.
  4. Repeat: Repeat this process, starting at the next book in line, until the entire bookshelf is sorted.

This analogy simplifies the Selection Sort process, making it easier to understand how the algorithm works.

void selectionSort(int arr[], int n)
{
	// The outer loop has to go over all n elements in order to compare them
	// and bring them to the front if they are the minimum value. Since
	// we start counting at 0, going from 0 to (n - 1) contains the first 
	// n elements.
	for (int i = 0; i < n; ++i)
	{
		// We initialize the 1st element as the index of the minimum:
		int indexMin = i;
		// We check all i + 1 indeces after i in order to check if there is
		// another index with a lower value:
		for (int j = i + 1; j < n; ++j)
			if (arr[j] < arr[indexMin])
				indexMin = j; // if so we make this the smallest index.
		
		// Once we have found the smallest index, we swap the values
		// using the "standard" C swap:
		int temp = arr[i];
		arr[i] = arr[indexMin];
		arr[indexMin] = temp;
	}
}

When analysing the time complexity of the selection sort algorithm, one first observes that with the inner loop in the function will on average compare approximately \(\frac{n}{2}\) before the lowest value of the iteration is found. This loop to find the lowest value is itself also run \(n\) times. Hence we can conclude that we have a time complexity of \(O(\frac{n}{2} \cdot n) = O(n^2)\).

Insertion sort#

Finally, let’s take a look at the Insertion Sort algorithm. This algorithm works slightly differently than the two algorithms mentioned earlier. Conceptually, the array is divided into two parts: the first part represents the sorted portion, while the second part consists of the elements that still need to be sorted. (This division is conceptual rather than physical.)

Using the bookshelf analogy one last time, the insertion sort algorithm can be described as follows:

  1. Start: Begin with the first book on the bookshelf (the book at index 0). This book is considered the first element of the “sorted” portion of the array.
  2. Next: Move to the second book (at index 1). If this book is larger than the first book, it is already in its correct place within the sorted portion. If it is smaller, move it to index 0, and shift the previous first book to index 1. Now the sorted portion contains the first two books in order.
  3. Iterate: Proceed to the third book (at index 2). Determine whether it is already in the correct place within the sorted portion (e.g., the thickest book in the sorted part) or whether it needs to be inserted before one of the earlier books. Continue this process, ensuring that each book is placed in its correct position within the sorted portion before moving on to the next book.

This step-by-step approach gradually builds the sorted portion of the array, ensuring all books (or elements) are in their correct positions by the end. Now, for the implementation, we go back to the drawing board (i.e., our C compiler):

void insertionSort(int arr[], int n)
{
    // Implementation-wise: we can start the outer for loop at the second element (i = 1),
    // and iterate over i until the last element (i = n - 1), since the first element
    // (index 0) is already considered sorted.
    for (int i = 1; i < n; ++i)
    {
        // Save the current index and value of the element that we want to insert
        // into its correct position in the sorted part of the array.
        int insertIndex = i;
        int insertValue = arr[insertIndex];

        // Use this iterator to check where in the sorted portion of the array
        // the current value belongs. Start with the last element of the sorted part.
        int j = i - 1;

        // While the previous element is larger than the current value, shift the
        // previous element one position to the right. This creates space for the
        // current value to be inserted into its correct position.
        while (j >= 0 && arr[j] > insertValue) {
            arr[j + 1] = arr[j]; // Shift the larger element one position to the right.
            insertIndex = j;     // Update the insertion index.
            j--;                 // Move to the next element in the sorted part.
        }

        // Insert the current value at its correct position in the sorted portion.
        arr[insertIndex] = insertValue;
    }
}

Once again, we are dealing with approximately \(\frac{n}{2}\) comparisons needed to sort each value within the sorted part. Since this sorting process is repeated for every element, we end up with a time complexity of \(O(\frac{n}{2} \cdot n) = O(n^2)\).
